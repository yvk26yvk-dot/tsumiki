対象: 実使用フィードバック4件(日付指定・タブ改名・月別アコーディオン・カレンダービュー)

## 決定事項(ユーザー承認済み。仮決めA/B/Cも設計提示時に承認)

| 論点 | 決定 |

| ①遡り範囲 | 無制限(未来のみ選択不可)。デフォルトは今日 |
| ①保存済み過去日 | 今日と同じ「完了画面→書き直す」の挙動(仮決めB) |
| ①深夜の確認 | 0:00〜4:59にきょうタブを開き昨日が未記入なら、コウテイちゃんが「昨日のぶん?」を一度だけ確認(セッション中記憶) |

## 1. 日次チェックの日付指定

- きょうタブ上部に**日付チップ**(「7月16日(木) ▾」風。今日以外を選択中は
  ローズ系で強調+「今日にもどる」が出る)。タップで月送りカレンダーを開閉。
- カレンダーは共通関数 `buildCalendar(year, month, opts)` を使用。
  opts: `{maxDate}`(①は今日まで)、`{dots}`(④用)、`{onPick(dateKey)}`。
  未来日はグレーアウト・タップ無効。
- 内部状態: モジュールレベル `const todayKey` の参照を `let activeDayKey` に置換。
  `renderDaily`/`renderDeck`/`finishDaily`/`kotei.daily`呼び出し周りが参照。
  **日次タブに入るたび今日にリセット**(switchViewの分岐で)。
- 保存済みの日 → 完了画面(見出しは過去日なら「この日の記録、ぜんぶ受けとってるよ」
  等、日付が分かる文言)+「書き直す」。未保存の日 → 空のスライドデッキ。
- **深夜0〜5時の確認**: きょうタブ表示時、`new Date().getHours()<5` かつ
  昨日(暦日-1)の記録が未保存なら、スライドの上の吹き出しを
  「もしかして、昨日のぶんかな?」+[昨日を書く/今日を書く]ボタンに差し替える。
  選択したらセッション中は再表示しない(`sessionStorage` 不使用、JS変数でよい)。
- 保存キーは `activeDayKey`。store層のsaveDaily(dateKey, entry)は元々任意キーを
  受けるため**store層の変更ゼロ**。同期(outbox)もそのまま動く。
- 深夜に「昨日を書く」で保存した場合も、日付チップは選択日を表示し続ける
  (保存完了画面にも日付を出す)。

## 2. タブ改名「ふりかえり」→「こんしゅう」

- nav のラベル、renderWeekly内の見出し/誘い文言、日次完了画面の
  「ふりかえりを見る」ボタン(→「こんしゅうを見る」)を改名。
- `switchView('weekly')`・`view-weekly` 等の内部識別子は変更しない。
- CLAUDE.mdのUI/UX原則の「ふりかえりタブは〜」の記述も「こんしゅう」に更新。

## 3. きろくの月別アコーディオン化

- 現在の月見出し(`.rec-month`)を開閉可能なヘッダーに変更:
  「2026年7月(12件) ▾」。件数は月内カード数(日次+週次)。
- **今月のみ初期展開**。開閉状態はJS変数(セッション中のみ)。
- **週次カードの統合**: 週次カード(Wキー)を「その週の月曜日が属する月」の
  アコーディオン内に、日次カードと日付降順で混在表示。週次カードには
  「週」バッジ(`.rec-tag`流用)を付ける。
- フィルタチップの挙動は現行踏襲(「週次」選択時は週次カードのみ、
  「すべて」で混在)。フィルタ変更時もアコーディオン構造・開閉状態を維持。

## 4. きろくのカレンダービュー

- きろく最上部に **[カレンダー | リスト]** セグメント切替(`.tags`流用の2ボタン)。
  初期はリスト。選択はJS変数(セッション中のみ)。
- リスト選択時: フィルタチップ+③のアコーディオン表示(現行の進化形)。
- カレンダー選択時: フィルタチップを隠し、月送りカレンダー(共通
  `buildCalendar`、`dots:記録がある日` オプション)を表示。
  - ドットは**日次記録がある日**にローズ(`--rose`)の点。
  - 日タップ → カレンダー直下にその日のカード(きろくと同じ項目レイアウト)。
    記録なしの日は「この日はおやすみだったんだね。それも大事な一日だよ」等の
    コウテイちゃん空状態。
  - 月送りは「‹ 2026年7月 ›」。未来月への移動は可(ドットが無いだけ)。

## 5. 共通・技術

- カレンダーグリッド生成は単一関数 `buildCalendar(year, month, opts)` で共用
  (①: maxDate=今日・選択コールバック / ④: dots・選択コールバック)。
  月曜はじまりではなく**日曜はじまり**(既存の週ストリップWD配列と整合)。
- 新規CSS: 日付チップ・カレンダーグリッド・アコーディオンヘッダー・セグメント
  切替。配色は既存変数のみ。
- sw.jsのCACHEはリリース時に `kotei-v7`。
- store層・kotei層・認証・設定画面・同期ロジックは一切変更しない。

## 6. 検証(完了条件)

1. ①今日のフロー(従来動作)が不変。日付チップから過去日を選び保存→きろくに
   反映→Supabase `daily` に該当date_keyの行→リロードで復元
2. ①未来日が選択不可。過去の保存済み日は完了画面→書き直すで編集できる
3. ①深夜帯(端末時刻を変えて)昨日未記入なら確認が出る。昨日記入済みなら出ない
4. ②タブと文言が「こんしゅう」になり、weeklyの機能は不変
5. ③今月だけ開いた状態で表示。開閉できる。週次カードが該当月に「週」バッジ付きで
   混在。フィルタも従来通り効く
6. ④切替でカレンダー⇄リスト。記録日にドット。日タップでカード表示・
   記録なし日は空状態。月送りできる
7. PWAで一連の回帰(ログイン・保存・同期・ログアウト)に影響なし

- Capacitorブランチへの反映 — 本スプリントのmainマージ後、ブランチ側でmainを
  取り込んで実機検証時に一緒に確認
